7. Terminology

In this document we will use the following definitions of terms:

Legacy RAG

Legacy retrieval augmented generation, which uses the industry established process of uploading documents to a text extraction solution, the text is parsed linearly, the text is chunked into segments with a specific equal length (for example 2,000 characters per chunk). The chunks are truncated based on common punctuation that ends sentences such as (".","!","?"). The text chunks have a character overlap between each chunk to ensure important context between chunks is not lost.

Blockify®

Blockify is the patented solution from Iternal Technologies that includes a data ingestion pipeline of the following steps: Uploading documents to a text extraction solution, the text is parsed linearly, the text is chunked into segments with a specific equal length (for example 2,000 characters per chunk). The chunks are truncated based on common punctuation that ends sentences such as (".","!","?"). The text chunks have a character overlap between each chunk to ensure important context between chunks is not lost. The chunks are processed by a specially developed LLM that is designed to extract the IdeaBlocks from the source chunks. The IdeaBlocks are distilled and deduplicated via another series of LLMs and clustering similarity algorithms, the process is repeated a number of times for optimal dataset distillation. The IdeaBlocks are made available for optional human review.

IdeaBlocks

An IdeaBlock is any self-contained concept or idea about a topic, for example your company, products, services, etc. An IdeaBlock is typically 2 - 3 sentences in length. It contains a Name, a Critical Question, a Trusted Answer, an Index, Tags, Keywords, and other metadata

Additional Terms

Term Definition
RAG (Retrieval-Augmented Generation) An AI architecture that combines information retrieval with language generation, using a knowledge base to provide context for LLM responses.
Vector Embedding A numerical representation of text in high-dimensional space (typically 1536 dimensions for OpenAI models), where semantically similar texts have similar embeddings.
Cosine Distance A measure of similarity between two vectors, calculated as 1 minus the cosine of the angle between them. Lower values indicate higher similarity.
Chunking The process of splitting documents into smaller pieces for processing. Traditional chunking uses fixed character/token counts; Blockify® uses semantic boundaries.
Distillation The Blockify® process of deduplicating and merging semantically similar IdeaBlocks to eliminate redundancy while preserving information completeness.
Critical Question The specific question that an IdeaBlock is designed to answer. This field helps vector search match user queries to relevant content.
Trusted Answer The validated, complete answer contained within an IdeaBlock. Trusted answers are extracted and verified by the Blockify® LLM.
Enterprise Duplication Factor A multiplier ({{ config.enterprise_dup_factor }}X) representing typical data redundancy in enterprise environments, based on industry research showing 8:1 to 22:1 duplication ratios.
Token The basic unit of text processing for LLMs, approximately 4 characters on average. Token count directly affects API costs and processing time.
